question:
    1. 指针和引用区别
    2. const int* p/ int* const p/ const int* const p
    3. 函数参数什么时候用引用
    4. static 局部变量
    5. static 全局变量
    6. static 类成员变量 / 成员函数



answer:
    1.
        指针占用内存空间，是独立的变量；引用是变量别名，不开辟内存，和原变量用同一个地址
        指针可以是空指针；引用必须绑定对象，不能为空
        指针可以不初始化；引用必须初始化
        指针可以改变指向；引用绑定之后不能再改绑
        指针需要*解引用，用->访问成员；引用像普通变量一样使用
        指针变量占用内存；引用语义上是别名，编译器实现可能是指针
        指针可以多级，引用没有“引用的引用”这种普通用法
    2.
        以*为分割，const离int近，锁的是*p，代表不能通过这个指针修改它指向的内容；const离p近锁的是p，代表指针变量不能修改，也就是说指针不能指向其他变量；两边都有const就是两个都不能改
    3.
        参数有对象，用引用；可能没有对象，用指针
        只读大对象，用const 引用 ，比如 void printname(const stu &s){}
        要修改实参，用普通引用
    4.
        定义在函数里的static局部变量，生命周期不是单词函数调用，而是整个程序运行期间
        作用域是局部的，生命周期是全局的
    5.
        定义在函数外边，但只能被当前源文件使用的全局变量
        普通全局变量可以在其他文件通过extern使用，static 全局变量不行
        程序开始时创建，程序结束时销毁
    6.
        static 类成员变量不是属于某个对象的，是属于类的，或者说是所有对象共享的，一般需要类外定义一次
        static 成员函数也是属于类的，不是属于对象的，不能访问普通成员变量
