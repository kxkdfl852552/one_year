questions：
    1. 左值 / 右值
    2. 拷贝构造
    3. 移动构造
    4. std::move
    5. 为什么 move 可以减少拷贝

answers：
    1.拷贝移动
        拷贝：复制资源，源对象继续拥有资源
        移动：转移资源，源对象放弃原资源
    2.左值右值
        左值：有明确身份，有名字，之后还能继续访问的对象
        右值：临时值，或者被明确标记为可以去走资源的对象
    3.三种常见引用
        普通左值引用
            Tensor& ref = a;
        常量左值引用，既能绑定左值，也能绑定临时对象，而且不会拷贝
            const Tensor& ref = a;
            const Tensor& ref = Tensor(1024);
        右值引用,主要用于绑定右值
            Tensor&& ref = Tensor(1024);
            Tensor&& ref = std::move(a);
    4.拷贝构造函数
        使用一个已有对象，创建一个新的同类型对象
        形式 Tensor(const Tensor& other);
        使用方法
            Tensor a(1024);
            Tensor b = a;  //等号右侧是同类型左值、左侧是未初始化新对象时，= 只是语法糖，编译器不会先生成空对象再赋值，而是直接用拷贝构造初始化。
            Tensor c(a);
        为什么参数是const Tensor& other
            因为不修改源内容，所以用const；如果不用引用，源对象传给other，本身就需要复制一次，复制又要调用拷贝构造，就会递归调用
        浅拷贝
            编译器简单地逐个复制成员 data_ = other.data_;
            a.data_ 和 b.data_会指向同一内存，析构时会double free
        深拷贝
            正确的拷贝构造要重新申请内存
            Tensor(const Tensor& other):
                size_(other.size_),
                data_(other.size_ == 0 ? nullptr : new float[other.size_]){
                    std::cout << "copy constructor\n";
                    for (size_t i = 0; i < size_; ++i) {
                        data_[i] = other.data_[i];
                    }
                }
    5.移动构造函数
        Tensor(Tensor&& other) noexcept;
        使用一个临时对象或明确不再需要原内容的对象，创建新对象。
        Tensor a(1024);
        Tensor b = std::move(a);

        Tensor(Tensor&& other) noexcept:
            size_(other.size_),
            data_(other.data_){
                std::cout << "move constructor\n";
                other.size_ = 0;
                other.data_ = nullptr;
            }
        一定要置空原指针，不然会double free
        移动不是让两个对象共享资源，而是新对象接管资源，旧对象放弃资源。
    6.std::move
        std::move本身不会移动任何数据
        std::move(a);表达的是我允许后续代码可以把a当作可以取走资源的对象
        Tensor b = std::move(a); 真正的移动在Tensor的移动构造里
        std::move不保证一定移动，在没有移动构造，只有拷贝构造的时候，也能匹配到const Tensor&
        std::move 只是让移动操作有机会被选中，不保证一定发生移动。
    7.右值引用变量本身是左值
        void func(Tensor&& tensor) {
            Tensor a = tensor;
        }
        这里的Tensor&& tensor虽然是右值引用，但是tensor是左值，Tensor a = tensor会调用拷贝构造
        正确的移动构造为
            void func(Tensor&& tensor) {
                Tensor a = std::move(tensor);
            }
    8.构造和赋值
        构造，目标对象还不存在
            Tensor a(100);
            Tensor b = a;
            调用拷贝构造 Tensor(const Tensor& other);
            Tensor a(100);
            Tensor b = std::move(a);
            调用移动构造 Tensor(Tensor&& other);
        赋值，目标对象已经存在
            Tensor a(100);
            Tensor b(200);
            b = a;
            调用拷贝赋值  Tensor& operator=(const Tensor& other);
            Tensor a(100);
            Tensor b(200);
            b = std::move(a);
            调用移动赋值  Tensor& operator=(Tensor&& other);
    9.拷贝赋值
        拷贝赋值不仅要复制数据，还要处理当前对象已有的资源
        要先申请新内存，再处理当前的已有资源，最后替换
    10.移动赋值
        检查是否自己移动给自己
        释放自己原来的资源
        接管 other 的资源
        把 other 清空
        返回 *this
    11.为什么移动构造里通常加noexcept
        因为标准容器例如 vector 在扩容迁移元素时，为了异常安全，会优先使用 noexcept 的移动构造。如果移动构造可能抛异常，vector 可能退回使用拷贝构造，避免移动过程中破坏原对象。
        移动操作通常只是转移指针、句柄等资源所有权，不需要申请新资源，因此一般不会失败，所以适合声明 noexcept。
        但是如果移动过程中有可能发生异常，就不能强行声明 noexcept，否则异常会导致 std::terminate。
    12.移动后的对象还能不能使用
        移动后原对象必须仍然能安全析构、重新赋值、保持有效状态，标准库通常说处于有效但未指定，不要用size去判断
        实际工程习惯：对象被移动后，只让它析构、重新赋值，或者调用明确支持移动后状态的函数。
    13.对 const 对象使用 std::move，通常还是拷贝。
    14.返回对象时怎么写
        Tensor create_tensor() {
            Tensor result(100);
            return result;
        }
        编译器通常可以进行 NRVO：
            函数返回一个局部对象时，按原始逻辑要经历三步：
                在函数栈上创建局部对象；
                调用拷贝构造，把对象复制到调用方外层内存；
                销毁函数内局部对象（调用析构）。
            大对象（容器、Buffer、自定义类）拷贝开销极大，NRVO 直接把这三步合并，完全消除拷贝。
        显式 std::move 可能阻止 NRVO。
        返回局部对象时，一般直接 return result;
        即使没有发生 NRVO，编译器也通常可以对局部返回变量使用隐式移动
